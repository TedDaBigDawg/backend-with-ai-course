# Final Capstone Project

**Proposal Due:** Start of Week 13  
**Project Due:** End of Week 14  
**Weight:** 25%

## Brief

Build a complete, production-ready backend API that demonstrates all the skills you've learned throughout the course. This should be a substantial project that you can showcase in your portfolio.

## Requirements

Your final project must include:

### Core Requirements
- **NestJS Framework:** Properly structured application
- **Database:** PostgreSQL with Prisma ORM
- **Authentication:** JWT-based authentication with role-based access
- **API Documentation:** Complete Swagger/OpenAPI documentation
- **Validation:** Input validation using DTOs
- **Error Handling:** Comprehensive error handling
- **Multiple Resources:** At least 2-3 related resources/entities
- **Relationships:** Database relationships (one-to-many, many-to-many)

### Optional but Recommended
- File upload functionality
- External API integration
- Pagination and filtering
- Environment configuration
- Deployment to production platform

## Project Ideas

Choose one of these or propose your own:

1. **Task Management API:** Users, tasks, projects, teams
2. **E-commerce Backend:** Products, orders, customers, cart
3. **Blog Platform API:** Posts, comments, users, categories
4. **Social Media API:** Posts, users, likes, follows, comments
5. **Learning Management System:** Courses, students, instructors, assignments

## Deliverables

1. **GitHub Repository:**
   - Complete source code
   - Well-organized structure
   - README with setup instructions
   - Clear commit history

2. **README Documentation:**
   - Project description and purpose
   - Features list
   - Setup instructions
   - API documentation link
   - Technology stack
   - Future improvements

3. **API Documentation:**
   - Swagger/OpenAPI documentation
   - All endpoints documented
   - Examples provided
   - Authentication guide

4. **Presentation (Week 14):**
   - 5-10 minute presentation
   - Live demo of API
   - Showcase key features
   - Discuss challenges and learning

5. **Reflection (Optional):**
   - What you learned
   - Challenges overcome
   - How AI assisted your learning
   - Future improvements

## Rubric (25 points)

- **Functionality (8):** All features working, CRUD operations, authentication, relationships
- **Code Quality (5):** Well-structured, follows best practices, proper error handling
- **Documentation (4):** Complete README, Swagger docs, clear instructions
- **Architecture (4):** Proper NestJS structure, organized modules, separation of concerns
- **Testing (2):** Tested with Postman, endpoints working correctly
- **Presentation (2):** Clear presentation, demonstrates understanding
