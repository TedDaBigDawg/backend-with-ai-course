# Mini Project 3 — API with Documentation

**Due:** End of Week 09  
**Weight:** 15%

## Brief

Add comprehensive API documentation to your existing API (from Mini Project 1 or 2) using Swagger/OpenAPI. Ensure all endpoints are documented with descriptions, request/response examples, and authentication information.

## Requirements

- Swagger/OpenAPI setup in NestJS
- All API endpoints documented
- DTOs documented with descriptions and examples
- Authentication documented (if applicable)
- Interactive Swagger UI accessible
- Request/response examples included
- Proper API tags and organization

## Deliverables

- GitHub repository link with source code
- README with:
  - Swagger documentation URL
  - How to access Swagger UI
  - API overview
- Swagger UI accessible (locally or deployed)

## Rubric (15 points)

- **Swagger Setup (3):** Swagger properly configured and accessible
- **Endpoint Documentation (4):** All endpoints documented with descriptions
- **DTO Documentation (3):** All DTOs documented with examples
- **Examples & Clarity (2):** Clear examples and descriptions
- **Organization (2):** Well-organized with tags
- **Completeness (1):** All features documented

## Prerequisites

- Completion of Mini Project 1 or 2
- Understanding of Swagger from Week 09

## AI-Assisted Learning

- Use AI to generate Swagger decorator examples
- Ask AI to help write clear API descriptions
- Use AI to review your documentation for completeness
- Have AI suggest improvements for documentation clarity

## Submission

- Update your PR or create a new one: "MP3: Your Name — Project Title"
- Include repository link and Swagger documentation URL
